## 2. 关键数据流

**① 实时数据（推送主链路）**
EMS → `POST /north/online_data_push` → 采集层解析 → 写最新值到 Redis + 发 Pub/Sub → WebSocket 网关推前端；同时落原始时序表 → 连续聚合自动降采样。采集层把测点值喂给规则引擎。

**② 预警**
规则引擎消费实时值 → 命中阈值 → 去抖计时(continuous_time) → 产生告警(写 alarms+alarm_events) → 检查抑制(屏蔽/维护窗口) → 防轰炸合并 → 通知层按级别路由发送。值恢复 → 自动恢复 → 恢复通知。

**③ EMS 设备级告警**
EMS → `POST /north/online_alarm_push` → 告警接入层 → 过滤仅保留 event_type∈{0,21,30} → 去重 → 入 alarms(source=ems) → 通知。

**④ 历史查询/回补**
前端 `/history/query` → 历史模块按范围选层（短→原始表，长→5min 聚合）→ 返回。
推送中断检测 → 回补任务获取锁 `lock:history` → 按测点≤100 分批、按≤1 天分片、串行调用 `offline_value` → 写降采样层 → 标记缺口已补。

**⑤ 连接保活**
连接管理常驻：login→存 token→每 20s heart→失败/106/2 错误码→退避重连→重订阅 data+alarm；状态写 `ems:conn`。

## 3. 后端模块划分（FastAPI）

| 模块 | 职责 |
|---|---|
| `api/` | REST 路由（按 A1–A8 分文件）+ 鉴权依赖 + WebSocket 网关 |
| `ems/client.py` | EMS 客户端：封装 12 个北向调用，统一包头/token/错误码处理 |
| `ems/push_server.py` | 接收 EMS 推送的两个端点（data/alarm），挂载在 recv 地址 |
| `ems/connection.py` | 连接生命周期：登录、心跳任务、重连、订阅、状态维护 |
| `sync/config_sync.py` | 空间/设备/测点同步、增量比对、元数据关联 |
| `ingest/realtime.py` | 推送解析、最新值缓存、Pub/Sub、落库、喂规则引擎 |
| `history/query.py` | 历史查询路由选层 |
| `history/backfill.py` | 断连回补（分批/分片/串行/锁） |
| `engine/rules.py` | 规则评估、多档阈值、去抖、自动恢复 |
| `engine/lifecycle.py` | 告警状态机、生命周期事件、受理/确认 |
| `engine/suppress.py` | 屏蔽、维护窗口、防轰炸合并判定 |
| `notify/dispatcher.py` | 级别路由、接收人解析、重试、发送记录 |
| `notify/channels/` | 各渠道适配器：sms(内部短信平台)/email/dingtalk/wecom/voice/webhook |
| `reports/` | 告警统计、导出、定时报表生成 |
| `scheduler.py` | APScheduler：定时同步、回补巡检、报表、降采样维护 |
| `core/` | 配置(.env)、DB 会话、Redis、加密、日志、安全(JWT/RBAC) |
| `models/` & `schemas/` | ORM 模型(SQLAlchemy) 与 Pydantic 模型 |

## 5. Docker Compose 服务

```yaml
services:
  db:          # postgres + timescaledb 镜像 (timescale/timescaledb:latest-pg15)
  redis:       # redis:7
  backend:     # FastAPI (uvicorn)；依赖 db/redis；暴露 8000；同时承载 /north 推送端点
  frontend:    # 构建产物由 nginx 提供；反代 /api 与 /ws 到 backend
  # ems_mock:  # 联调用，可选启用，模拟共济EMS，暴露 9000
```

要点：
- backend 容器需同时对外暴露平台 API 端口与 EMS 推送接收端口（recv_port 指向 backend），保证 EMS/mock 能 POST 进来。
- 内网单机；提供 `make backup` / 文档化 `pg_dump` 定时备份。
- 时区统一（容器 TZ + DB），日志按模块分类输出。

## 6. 前端要点
- 实时刷新统一走 `useWebSocket`；断线自动重连。
- 趋势图 ECharts，大数据量用降采样层 + 按需加载。
- 大屏独立路由、深色主题、`rem`/`vw` 自适应、支持轮播；移动端对大盘/告警中心/详情做响应式断点。
- 角色驱动菜单与按钮级权限（readonly 隐藏写操作）。
